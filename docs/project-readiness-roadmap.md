# Project Readiness Roadmap

This document defines how Content Engine explains project readiness.

## Purpose

Readiness must be actionable. A percentage alone is not enough. The UI should show what is complete, what is missing, which category it belongs to, and the next action needed.

## Readiness categories

Readiness is split into three groups:

1. Structure readiness
2. Content readiness
3. Export readiness

## Checklist items

A project readiness checklist should include:

- project metadata
- folder structure
- idea intake
- canon seed
- character seed
- story outline
- production package schema
- voice package readiness
- visual package readiness
- export profile readiness
- review and approval status

## Score rule

The readiness percent is the number of complete checklist items divided by the total checklist items.

## UX rule

Every incomplete item should include a clear next action, such as:

- Create project scaffold
- Load idea intake
- Generate outline
- Create voice package
- Approve draft artifact
- Select export profile

## Important distinction

Project readiness is not the same as final release readiness. A project can be structurally ready while still needing rights, quality, and final release review.
