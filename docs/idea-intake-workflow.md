# Idea Intake Workflow

This document defines how a user loads raw story ideas into Content Engine.

## Purpose

Idea intake is the bridge between a rough concept and structured project context. Raw ideas are captured as draft material first. They do not become canon until reviewed and approved.

## Initial implementation path

The first workflow supports paste-in text:

1. Select or create a project.
2. Paste raw idea notes.
3. Add optional tone or direction.
4. Create idea intake draft.
5. Review the structured intake output.
6. Promote selected items later through canon, character, outline, and readiness workflows.

## Draft output sections

A structured intake draft should include:

- summary
- genre and tone candidates
- theme candidates
- character candidates
- canon candidates
- story arc candidates
- artifact roadmap
- next steps

## Canon rule

Idea intake output is draft context. Nothing from intake becomes canon automatically.

## Future import options

Later implementations may support file import, transcript import, markdown upload, and multi-fragment idea boards.
