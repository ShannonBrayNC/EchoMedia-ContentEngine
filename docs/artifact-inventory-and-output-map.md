# Artifact Inventory and Output Map

This document defines the project artifact inventory for Content Engine.

## Purpose

Readiness must map to concrete outputs. Users need to see what exists, what is missing, what is required, and what can be previewed, reviewed, regenerated, or exported.

## Inventory categories

Artifacts should be grouped by category:

- idea intake
- canon
- characters
- story outline
- manuscript
- screenplay
- production package
- visual prompts
- voice packages
- timelines
- export manifests
- release package

## Artifact states

Each artifact should show a text state:

- missing
- planned
- draft
- review
- approved
- exported
- released
- stale
- superseded

## Required versus optional

Required artifacts should be marked clearly. Optional artifacts can still be listed, but they should not block readiness unless selected as part of the project target format.

## Actions

An inventory item may expose actions:

- preview
- review
- generate
- regenerate
- export

## 100% readiness map

At project readiness 100%, the inventory should show which concrete outputs exist and which are approved. This does not replace final release readiness or rights review.
