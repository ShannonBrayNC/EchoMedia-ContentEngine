# Book 1 ElevenReader Checklist

Purpose: produce a single clean narration manuscript for ElevenLabs ElevenReader so Book 1 can be listened to without consuming generation credits or fighting chapter-level edits.

## Required structure

- [ ] Book title appears once at the top.
- [ ] Book title is bolded.
- [ ] Preface heading is present and bolded.
- [ ] Chapters appear in the correct order.
- [ ] Every chapter heading is bolded.
- [ ] Each major section begins cleanly with no build metadata.
- [ ] File contains only text meant to be read aloud.

## Remove before upload

- [ ] Repository paths.
- [ ] Export timestamps.
- [ ] Manifest instructions not intended for the reader.
- [ ] Source notes.
- [ ] Correction logs.
- [ ] Internal comments.
- [ ] Screenplay slug lines.
- [ ] Character labels from screenplay format unless converted into prose.
- [ ] Markdown table of contents unless it improves narration.
- [ ] Any note that starts to sound like a production artifact instead of a book.

## Formatting rules

- [ ] Use consistent heading style.
- [ ] Avoid decorative symbols that may be read aloud awkwardly.
- [ ] Use page breaks or clear spacing between chapters.
- [ ] Keep chapter titles short and readable.
- [ ] Convert Markdown emphasis to Word bold formatting in the `.docx` export.

## Listening review pass

During ElevenReader listening, capture:

- [ ] Awkward sentences.
- [ ] Repeated phrases.
- [ ] Missing transitions.
- [ ] Confusing speaker references.
- [ ] Continuity issues.
- [ ] Pacing drops.
- [ ] Places where prose sounds like notes instead of story.
- [ ] Places where Lantern/system language needs clarification.

## Done means

- [ ] Book 1 can be listened to from start to finish without narration artifacts.
- [ ] All corrections from the listening pass are logged.
- [ ] Canonical manuscript is updated after listening edits.
- [ ] KDP package can begin from the corrected manuscript.
