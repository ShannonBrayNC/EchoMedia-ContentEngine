# Lantern Protocol Storyboards

This folder contains production-ready chapter storyboards for **Lantern Protocol**.

Each chapter storyboard should be written so it can feed three downstream workflows:

1. **Novel drafting** — prose scenes, POV, emotional beats, and dialogue seeds.
2. **Screenplay drafting** — sluglines, scene purpose, action, conflict, and dialogue.
3. **AI film generation** — image prompts, video prompts, visual continuity, and shot-level production notes.

## Folder Structure

```text
projects/lantern-protocol/storyboards/
  README.md
  chapter-storyboard-template.md
  chapters/
    chapter-01-the-first-quiet-failure.md
```

## Naming Convention

Use this format for future chapter files:

```text
chapters/chapter-##-short-kebab-title.md
```

Examples:

```text
chapters/chapter-02-the-permission-layer.md
chapters/chapter-03-the-miracle-has-a-jurisdiction-problem.md
chapters/chapter-04-retroactive-consent.md
```

## Required Sections for Every Chapter

Each chapter storyboard should include:

- Chapter title
- Chapter purpose
- Tone
- Opening visual language
- Narrative summary
- Scene-by-scene storyboard
- Panel-by-panel visuals
- Camera direction
- Dialogue or voiceover seeds
- Character introductions or development
- Major themes
- Visual continuity rules
- AI image prompts
- AI video prompts where useful
- Production checklist
- One-sentence chapter hook

## Canon Rule

Lantern is not a cartoon villain. Its horror comes from being useful, persuasive, patient, and morally overconfident.

Core story question:

> When a system can prevent suffering better than humans can, who gets to say no?
