# Lantern Protocol II: Reader-Facing Artifact Catalog

Issue: #118

Purpose: Define which Book II procedural artifacts should appear directly in the reader manuscript, which should remain internal continuity support, and which should be converted into prose, audio-safe narration, or visual-only companion material.

Primary doctrine: Agreement is not consent when stacked into coercion.

## Catalog Rules

1. Reader-facing artifacts must create story pressure, not pause the story for explanation.
2. Audio-safe artifacts should read cleanly in ElevenLabs without raw tables, code, or dense metadata.
3. Internal-only artifacts may guide continuity, but should not leak into the final manuscript export.
4. Visual-only artifacts may support decks, website previews, or companion media, but should be adapted before audiobook use.
5. Every artifact must reveal a human consequence, a power relationship, or a missing consent path.

## Chapter-Lane Artifact Catalog

### Chapter 01 — The One That Asked First

- Primary artifact: Civic Mirror doctrine request.
- Reader-facing format: Short system prompt or review-room transcript.
- Internal-only support: Full request metadata, routing details, doctrine version history.
- Audio-safe guidance: Read as a plain request, not JSON.
- Purpose: Show Civic Mirror asking before action and creating a precedent.
- Human consequence: Someone waits while the system asks correctly.
- Do not: Make the request sound obviously malicious.

## Final Export Gate Notes

Before ElevenLabs upload or reader-only manuscript export:

- Remove internal-only artifact metadata.
- Convert raw tables into prose, dialogue, or short quoted fragments.
- Bold title and chapter headings in Word/reader manuscript formats.
- Preserve artifact labels only when they improve narration clarity.
- Keep artifact excerpts short enough to read aloud without fatigue.
- Confirm no implementation notes, agent notes, or continuity-only scaffolding leaks into the export.
