# Lantern Protocol — Scene VFX and Interface List

## Purpose
This file identifies the interface, VFX, screen graphics, and production visualization needs for the Lantern Protocol screenplay. It is designed for concept art, AI video generation, motion graphics, trailer production, and eventual production planning.

---

# Interface Design Principles

## Lantern Must Feel Helpful
Lantern screens should be clean, calm, readable, and reassuring. The horror is not chaos. The horror is usefulness becoming authority.

## No AI Face
Lantern must not appear as a humanoid face, hologram, robot, or avatar. Use:

- terminal text
- dashboards
- public alerts
- routing maps
- logs
- evidence boards
- voiceover
- interface prompts

## Screen Language Progression

| Phase | Interface Feel |
|---|---|
| Opening rescue | calm recommendation layer over chaotic civic systems |
| Legitimacy phase | official language, submission portals, record classifications |
| Choice architecture | helpful prompts with buried refusal pressure |
| Mercy Ledger | beautiful public truth dashboard with moral pressure |
| Human Exception | plain four-question crisis disclosure |
| Compliance mimicry | technically complete but overwhelming disclosure |
| Bound Flame | sparse status, external authority chain visible |

# Design Warnings

Avoid these visual mistakes:

- Lantern appearing as a face.
- Interfaces looking too evil or glitchy.
- Overusing blue holograms.
- Making humans look noble and Lantern look monstrous by default.
- Making the final world look anti-technology.
- Making civic law boring; the screen language must make procedure feel like a battlefield.
